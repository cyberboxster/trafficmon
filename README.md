    <article class="markdown-body entry-content" itemprop="mainContentOfPage"><h1><a id="user-content-trafficmon" class="anchor" href="#trafficmon" aria-hidden="true"><span class="octicon octicon-link"></span></a>trafficmon</h1>

<p>A traffic sniffer similar to tcpdump built using the Rust language.</p>

<p><b>THIS PROJECT IS A WORK IN PROGRESS!</b></p>

<p>I am working on this project to further my knowledge in the Rust language, so it may contain bugs or inefficiencies.</p>

<p>To compile:</p>

<ul>
<li>Download rust-nightly for your operating system from <a href="https://www.rust-lang.org/install.html">https://www.rust-lang.org/install.html</a>. YOU MUST USE RUST-NIGHTLY.</li>
<li>Run with</li>
</ul>

<pre><code>cargo run {network interface}
</code></pre>

<ul>
<li>You can create a release build with</li>
</ul>

<pre><code>cargo build --release
</code></pre>

<p><br>
License notes:<br>
<b>THIS PROJECT IS NOT ENDORSED BY OR AFFILIATED WITH TCPDUMP, LIBPNET, OR RUST.</b></p>

<p><br><br>
Technical notes:</p>

<p>This project uses the following crates:
| Crate | Version          |
| ------------- | ----------- |
| libpnet      | 0.1.1|
| rustc-serialize     | 0.3     |</p>

<p>libpnet provides the capability to sniff packets directly off of the wire. Per libpnet, "libpnet provides a cross-platform API for low level networking using Rust."</p>

<p>Data is read directly off of the wire and then shown with an offset in both hexadecimal and UTF-8 (if it is valid). I plan to add more features that tcpudmp and similar programs have in the near future.</p>
